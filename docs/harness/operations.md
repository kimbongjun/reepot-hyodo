# Operations

## 일상 운영 체크

- 관리자 페이지 접속 가능 여부
- 새 댓글 유입 여부
- 좋아요 수 증가 여부
- 유튜브 URL 노출 상태
- CSV 다운로드 가능 여부
- UTF-8 언어셋 체크

## 장애 대응 시작점

### 1. 빌드/배포 문제

- 최근 변경이 UI인지 서버 로직인지 스키마인지 먼저 분리한다.
- 동일 코드 기준으로 `npm run build`를 다시 실행한다.
- 환경변수 누락 여부를 가장 먼저 확인한다.

### 2. 댓글 등록 실패

확인 순서:

1. `/api/comments` 응답 코드 확인
2. `SUPABASE_SERVICE_ROLE_KEY` 존재 확인
3. `comment_submissions` 테이블 쓰기 가능 여부 확인
4. 요청 본문 검증 실패 메시지 확인

### 3. 좋아요 실패

확인 순서:

1. `/api/comments/[id]/like` 응답 확인
2. `increment_public_comment_like` 함수 존재 확인
3. `public_comments.like_count` 컬럼 존재 확인

### 4. 관리자 접근 실패

확인 순서:

1. 로그인 세션 존재 확인
2. 로그인 계정 이메일 확인
3. `ADMIN_EMAILS` 설정 확인
4. `/api/admin/settings` 또는 `/api/admin/export`의 `401` 여부 확인

### 5. 설정 저장 실패

확인 순서:

1. 관리자 권한 확인
2. `site_settings` 테이블 존재 확인
3. upsert 충돌 키가 `setting_key` 기준으로 동작하는지 확인

## 변경 작업 시 운영 메모

특히 아래 변경은 운영 메모를 남기는 편이 좋다.

- 환경변수 변경
- Supabase 스키마 변경
- 관리자 허용 이메일 변경
- 공개 폼 유효성 검증 변경
- 좋아요 로직 변경

## 향후 보강 권장 항목

- 자동 테스트 추가
- CI 검증 단계 추가
- 스키마 마이그레이션 체계 도입
- 배포 플랫폼별 실제 명령 문서화
