# Deployment

## 배포 전 전제

이 문서는 특정 호스팅 도구 명령보다 "배포에 필요한 상태"를 먼저 정의한다.

배포 전 아래가 준비되어 있어야 한다.

- 로컬 또는 CI에서 `npm run build` 통과
- 대상 환경의 환경변수 등록 완료
- 최신 `supabase/schema.sql` 반영 완료
- 관리자 허용 이메일 확인 완료

## 배포 전 체크리스트

- `NEXT_PUBLIC_SUPABASE_URL` 설정
- `NEXT_PUBLIC_SUPABASE_ANON_KEY` 설정
- `SUPABASE_SERVICE_ROLE_KEY` 설정
- `ADMIN_EMAILS` 설정
- 댓글 저장 흐름 확인
- 좋아요 증가 흐름 확인
- 유튜브 설정 저장 흐름 확인

## 권장 배포 순서

1. 코드 변경 내용을 확정한다.
2. `npm run build`로 빌드 성공을 확인한다.
3. 스키마 변경이 있으면 운영 Supabase에 `supabase/schema.sql`을 반영한다.
4. 배포 환경에 환경변수가 모두 등록되어 있는지 확인한다.
5. 애플리케이션을 배포한다.
6. 배포 직후 공개 페이지와 관리자 페이지를 스모크 체크한다.
7. 배포는 git commit, push까지 진행한다.

## 배포 후 확인

### 공개 기능

- 홈 페이지 렌더 정상
- 댓글 작성 가능
- 공개 댓글 반영 확인
- 좋아요 증가 확인
- 유튜브 섹션 노출 확인

### 관리자 기능

- 로그인 가능
- 대시보드 렌더 정상
- CSV 다운로드 가능
- 유튜브 URL 저장 가능

## 롤백 판단 기준

아래 중 하나라도 재현되면 즉시 롤백 또는 긴급 수정 후보로 본다.

- 댓글 등록 실패
- 관리자 로그인 후 대시보드 진입 불가
- 좋아요 API 실패
- 설정 저장 실패
- 빌드는 성공했지만 런타임에서 서버 오류 반복
